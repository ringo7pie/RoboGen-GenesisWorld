必須 — 全員が必ず行う 条件付き — 特定機能を使うときだけ 任意 — やると便利 場所: ローカルPC GitHub Colab OpenAI Drive

GitHub リポジトリの作成と push

必須 ローカルPCGitHub ⏱ 約 10 分

Colab のノートブックはセットアップ用スクリプトを GitHub から git clone して使います。そのため、作成済みのファイル一式を公開 (Public) リポジトリとして GitHub に置く必要があります。

  1. github.com/new で新規リポジトリを作成します。
    • リポジトリ名: RoboGen-GenesisWorld（変更可。変更した場合は手順2の URL も合わせる）
    • 公開範囲: Public（Colab の無認証 clone に必要）
    • 「Add a README file」等はすべてチェックなし（空のリポジトリにする）
  2. 現在のフォルダは社内の Azure DevOps リポジトリの内部にあるため、公開用フォルダへコピーしてから push します。PowerShell で以下を実行してください:
# 公開用フォルダへコピー（参照用の RoboGen クローンと Claude 設定は除外する）
robocopy E:\nkdx\USDFormat\Proj\RoboGen-GenesisWorld E:\nkdx\RoboGen-GenesisWorld-pub `
    /E /XD RoboGen .claude __pycache__ /XF CLAUDE.md

cd E:\nkdx\RoboGen-GenesisWorld-pub
git init
git add .
git commit -m "Initial commit: RoboGen + Genesis on Google Colab"
git branch -M main
git remote add origin https://github.com/<アカウント名>/RoboGen-GenesisWorld.git
git push -u origin main
push は必ずご自身の操作で実行してください（このプロジェクトの運用ルール上、Claude は git push を自動実行しません）。Claude Code のプロンプトで ! git push -u origin main のように ! を付けて実行することもできます。
先に手順2（URL 置換）を済ませてから push すると1回で済みます。push 後に置換した場合は再 push してください。
GitHub 上で notebooks / scripts / patches / docs などのフォルダが見えること。

ノートブック内の URL 置換

必須 ローカルPC ⏱ 約 2 分

両方のノートブックの「セル3」にあるリポジトリ URL のプレースホルダを、手順1で作った実際の URL に書き換えます。

  1. 対象ファイル（2つ）:
    • notebooks/01_genesis_demo.ipynb
    • notebooks/02_robogen_colab.ipynb
  2. 次の行の <YOUR_ACCOUNT> を GitHub アカウント名に置換します:
# 置換前
REPO_URL = "https://github.com/<YOUR_ACCOUNT>/RoboGen-GenesisWorld.git"
# 置換後の例（アカウント名が taro-yamada の場合）
REPO_URL = "https://github.com/taro-yamada/RoboGen-GenesisWorld.git"
アカウント名を Claude に伝えていただければ、この置換作業は Claude 側で実施できます（push はご自身で）。
2つのノートブックの REPO_URL に <YOUR_ACCOUNT> が残っていないこと。

ノートブック①を Colab で実行（Genesis デモ）

必須 Colab ⏱ 約 15 分

まず確実に動く Genesis 単体デモで、Colab 環境全体が機能することを確認します。API キーもアセットも不要です。

  1. ブラウザで次の URL を開きます（<アカウント名> は置換）:
    https://colab.research.google.com/github/<アカウント名>/RoboGen-GenesisWorld/blob/main/notebooks/01_genesis_demo.ipynb
  2. メニュー「ランタイム」→「ランタイムのタイプを変更」→ ハードウェア アクセラレータで「T4 GPU」を選択 → 保存
  3. 上からセルを順番に実行します（Shift + Enter）。初回は「このノートブックは Google が作成したものではありません」という警告が出るので「このまま実行」を選択
  4. セル5（セルフチェック）が全項目 OK になることを確認
  5. 最後のセルで Franka アームが動く動画 (mp4) が表示されれば成功
GPU を選択せずに実行すると、セル2で日本語のエラーメッセージが出て停止します。その場合はランタイム変更後、最初のセルからやり直してください。
アームが正弦波で揺れる 6 秒の動画がノートブック内に再生表示されること（マイルストーン M1 達成）。

ノートブック②をステップA まで実行（RoboGen 再生）

必須 ColabDrive ⏱ 約 30 分

RoboGen の環境構築（依存インストール＋パッチ適用＋アセット取得）を行い、例題タスク「ノート PC を開く」のシーン再構築動画を生成します。ここが最難関の関門で、ここさえ通れば環境構築は成功です。 OpenAI キーはまだ不要です。

  1. ①と同様に notebooks/02_robogen_colab.ipynb を Colab で開き、T4 GPU を選択
  2. セル2〜5 を順に実行（セットアップ一括 + セルフチェック。5〜10分）
  3. セル6 を実行すると Google Drive のマウント許可を求められます → 自分のアカウントを選択して許可
    • Drive を使いたくない場合は mount の行をコメントアウトしても動作します（アセットのキャッシュができなくなるだけ）
  4. 続けて PartNet-Mobility データセット（数 GB）が自動ダウンロードされます。SAPIEN サイトへのライセンス登録は不要です
  5. セル7（ステップA）を実行 → シーンの周回カメラ動画が表示されれば成功
ダウンロードで「Too many users have viewed or downloaded this file」エラーが出ることがあります（配布リンクへのアクセス集中）。その場合はブラウザで https://drive.google.com/file/d/1d-1txzcg_ke17NkHKAolXlfDnmPePFc6/view を開いて手動ダウンロードし、自分の Drive の MyDrive/robogen_data/partnet_dataset.zip にアップロードしてからセル6を再実行してください（キャッシュとして認識されます）。
初回実行でエラーが出る可能性が最も高いのはこのノートブックのセル4〜7です。エラーが出たら手順8の方法でログを共有してください。修正してすぐに再試行できます。
ノート PC のあるシーンを一周するカメラ動画が表示されること（マイルストーン M2〜M3 達成）。

OpenAI API キーの Colab Secrets 登録

条件付き OpenAIColab ⏱ 約 3 分

ステップB（LLM によるタスク新規生成）を使う場合のみ必要です。再生（A）と学習（C）だけならキー不要です。

  1. platform.openai.com/api-keys で API キーを用意（sk-... で始まる文字列）
  2. Colab 画面左側のカギアイコン 🔑（シークレット）をクリック
  3. 「＋ 新しいシークレットを追加」→ 名前欄に OPENAI_API_KEY（この名前ちょうどで）、値欄にキーを貼り付け
  4. その行の「ノートブックからのアクセス」トグルを ON にする
  5. ノートブック②のセル8 を実行 → 「疎通 OK」と表示されれば完了
タスク生成は GPT への問い合わせを複数回行うため API 課金が発生します（1タスクあたり数十円〜数百円程度が目安。使用モデルとプロンプト量で変動）。使用モデルはセル8の ROBOGEN_LLM_MODEL（既定 gpt-4o）で変更できます。
キーはノートブックにもリポジトリにも書き込まれません（Secrets → 環境変数経由で渡ります）。キーをコードに直接貼り付けないでください。
セル8 の出力に「疎通 OK: model=gpt-4o」と表示されること。
6

ステップB / C の実行（タスク生成・スキル学習）

条件付き ColabOpenAI ⏱ B: 約10分 ＋ C: 30分〜

ステップB（セル9）: 生成したいタスクを3つの変数で指定してから実行します。

  • TASK_DESCRIPTION — タスクの説明文（英語で。例: "open the door of the storage furniture"）
  • TASK_OBJECT — 対象オブジェクトのカテゴリ名（例: StorageFurniture, Microwave, Laptop）
  • TASK_OBJECT_ID — PartNet-Mobility のオブジェクト ID（例: 45374）

実行すると Objaverse 検索用の埋め込みデータ（数 GB）が追加ダウンロードされた後、LLM がタスク config・サブステップ分解・報酬関数コードを生成し、YAML の中身が表示されます。

ステップC（セル10〜11）: そのまま実行すると例題タスクを短縮設定（2万ステップ、約30分）で学習します。

  • ステップB で生成したタスクを学習したい場合 → セル10 の target = TASK_CONFIG を target = GENERATED_CONFIG に変更
  • 本格的に成功するまで学習したい場合 → --timesteps 20000 を --timesteps 1000000 などに増やす（数時間〜。Colab 無料枠のセッション上限に注意）
  • セル11 で結果動画の表示と Drive への保存が行われます
短縮設定はあくまで「パイプラインが最後まで走ることの確認」用です。2万ステップではタスク成功までは学習できません。
B: 生成された YAML がセル出力に表示される。C: 学習過程の動画が表示され、Drive の robogen_outputs に保存される（M4〜M5 達成）。
7

Google Drive の活用（キャッシュ・結果保存）

任意 Drive ⏱ 約 2 分

Colab はセッション終了で /content の中身がすべて消えます。Drive をマウントしておくと以下が永続化され、翌日の再開が速くなります。

  • アセット zip のキャッシュ → MyDrive/robogen_data/（数 GB。再ダウンロード不要になる）
  • 結果動画・学習成果 → MyDrive/robogen_outputs/<日時>/（セル11 が自動保存）
Drive 無料枠は 15GB です。PartNet-Mobility と Objaverse 埋め込みの両方をキャッシュすると数 GB 消費します。容量が厳しい場合はキャッシュなしでも動作します（毎セッション再ダウンロードになるだけ）。

エラーが出たときの報告方法

随時 ColabローカルPC ⏱ —

RoboGen の Colab 移植は初回実行での不具合修正を前提に設計しています（ローカル Windows では実行検証ができないため）。エラーが出たら次の2点をそのまま Claude に貼り付けてください。1往復で修正できる情報量になるよう、エラーメッセージに診断情報を自動で含めてあります。

  1. セルフチェックの結果表 — ノートブックのセル5（check_env.py）の出力全体。NG 項目には Python トレースバック全文が自動で付きます
  2. エラーが出たセルの出力全体 — 途中を省略せず、セル出力を最初から最後までコピー
修正は「Claude がスクリプト/パッチを修正 → あなたが再 push（または Colab 上で修正版を再実行）→ 再試行」の反復で進みます。よくあるエラーと自己解決手順は docs/troubleshooting.md にもまとめてあります。
