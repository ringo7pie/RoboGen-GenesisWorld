データフロー解説: 02_robogen_colab.ipynb

RoboGen フルパイプライン（ステップA 再生 / B タスク生成 / C スキル学習）が「何のファイルを読み、何を学習し、何のファイルを出力するか」の全体像（2026-07-07・Colab 実機検証済みの構成に基づく）

入力ファイル 処理（スクリプト） LLM（GPT）生成 学習（プランニング / 強化学習） 出力ファイル

パスの起点は Colab 上の RoboGen ディレクトリ /content/RoboGen/。<日時> は実行時刻（例 2026-07-07-03-37-35）、<タスク> はタスク名です。

2. 入力ファイル一覧（詳しい説明）

ファイル / パス説明
A. タスク定義（「何をするタスクか」を決める入力）
example_tasks/Open_Laptop/Open_Laptop_The_robotic_arm_opens_….yaml タスク定義 YAML（シーンの設計図）。登場オブジェクト（名前・種類・サイズ・机上配置・初期関節角）、タスク説明文、成果物の置き場（solution_path）を記述。例題は 46 種類同梱。ステップB を使うと GPT がこれと同形式の YAML を新規生成する
…/task_Open_Laptop/substeps.txt サブステップ分解。タスクを段階に分けた手順書（例:「grasp the laptop screen」→「open the laptop screen」）。1行 = 1 サブステップ
…/task_Open_Laptop/substep_types.txt 各サブステップの実行方式。primitive（モーションプランニングで解く: 把持・接近など）か reward（強化学習で解く: 操作など）かを1行ずつ指定
…/task_Open_Laptop/action_spaces.txt RL の行動空間。強化学習サブステップでロボットが出力する行動の種類（例 delta-translation = グリッパ先端の微小移動量）
…/grasp_the_laptop_screen.py
…/open_the_laptop_screen.py		 サブステップごとの実行・報酬コード（GPT 生成物）。primitive 用は「どのリンクを掴むか」の実行コード、reward 用は報酬関数 compute_reward()（例: 画面の開き角度が目標に近いほど高報酬）。強化学習はこの報酬を最大化するように進む
B. 3D アセット（シーンに登場する「物」のモデル）
data/dataset/<ID>/mobility.urdf
例: 9996=ノートPC, 45374=収納家具		 PartNet-Mobility データセット（自動ダウンロード）。蝶番や引き出しなど「動く関節」を持つ日用品の 3D モデル。URDF（ロボット記述形式）でリンク・関節・可動域を定義。RoboGen 公式のパース済み版を gdown で取得（ライセンス登録不要）。付属の関節ツリー・semantics テキストはステップB で GPT への入力にもなる
objaverse_utils/data/obj/<uid>/material_non_vhacd.urdf Objaverse の小物メッシュ（実行時に必要分だけ自動ダウンロード）。マグカップ・ペンなどタスクの脇役。タスク YAML 記載の uid から取得し、obj → 正規化 → URDF 化まで自動処理される
objaverse_utils/text_to_uid.json 物体名 → Objaverse uid の対応表（キャッシュ）。「a common coffee mug」等のテキストから 3D モデルを引くための辞書。ここに無い新規物体を探すときだけ、下の埋め込みファイルが必要になる
objaverse_utils/data/*.pt（任意） Objaverse 全 80 万件の文埋め込み（数十 GB・distractor 生成でのみ必要）。今回の検証ではディスク圧迫のため未使用（メインの学習には不要）
C. ユーザー入力・鍵（ステップB のみ）
TASK_DESCRIPTION / TASK_OBJECT / TASK_OBJECT_ID セル9 で指定する3変数。作りたいタスクの説明文（英語）・対象カテゴリ・PartNet ID。ここが「言葉からタスクを作る」入口
OPENAI_API_KEY（Colab Secrets） GPT-4o 呼び出し用の API キー。環境変数経由で渡され、ファイルには保存されない
D. 本リポジトリの実行スクリプト（処理の担い手）
scripts/setup_robogen.sh / apply_patches.sh / patches/*.patch RoboGen の clone（コミット固定）→ 依存インストール（requirements/robogen.txt、ompl==1.7.0 固定）→ Python 3.12 対応パッチ適用、の一括セットアップ
robogen_overrides/RL/sb3_learn.py / robogen_overrides/gym/__init__.py RL 学習部の stable-baselines3 実装（旧 ray 1.13 の置き換え）と、gym→gymnasium 互換 shim。RoboGen 内にコピーされて動く
scripts/run_robogen_task.py タスク実行の入口 CLI。--mode replay（再生）/--mode learn（学習）を切り替え、結果 GIF の mp4 変換まで行う

3. 「何から何を学習するのか」

学習サブステップの方式別に2種類の「学習」が走る（ステップC）
方式入力（何から）学習・出力（何を）
primitive
（OMPL モーションプランニング）
例:「画面を掴む」		 ・現在のロボット関節角
・シーン内の障害物形状（mobility.urdf 等の衝突形状）
・掴む対象リンク（grasp_the_laptop_screen.py が指定）		 厳密には「学習」ではなく探索計画。BITstar アルゴリズムが衝突しない関節軌道を数秒で計算し、実行後の状態を state_*.pkl に保存。次のサブステップの開始状態になる
reward
（SAC 強化学習・SB3）
例:「画面を開く」		 ・観測: ロボットと対象物の状態ベクトル（関節角・グリッパ位置・対象関節の開き角など）
・行動: action_spaces.txt の定義（delta-translation）
・報酬: open_the_laptop_screen.py の compute_reward()（GPT が書いた報酬式）
・開始状態: 直前サブステップの state_*.pkl		 方策ニューラルネットワーク（MLP 256×3層）を学習。「今の観測でグリッパをどう動かせば報酬（＝画面が開く度合い）が最大になるか」を試行錯誤（今回 2 万ステップ、本格学習は 100 万）。成果物は best_model.zip（最良方策）と latest_model.zip（最新）。5000 ステップごとに評価ロールアウトを行い eval_N/execute.gif と best_state/best.gif を記録
つまり本システムの「学習」の流れは — GPT がタスクと報酬関数（.py）を書く（ステップB）→ その報酬を使って SAC がロボットの動かし方（ニューラルネット方策）を学ぶ（ステップC） — という2段構えです。検証時のリターン（累積報酬）が負のままなのは 2 万ステップの短縮設定のためで、パイプライン確認としては正常です。

4. 出力ファイル一覧（詳しい説明）

出力の起点 = タスク YAML 内の solution_path（例題は example_tasks/Open_Laptop/task_Open_Laptop/、生成タスクは data/generated_task_from_description/…/task_<タスク>/）

ファイル / パス説明
ステップA（再生）の出力
<solution_path>/blip2/<日時>/construction.gif / .mp4 シーン構築の周回カメラ動画。組み上がったシーン（ノートPC・机・小物・ロボット）を 360° 撮影したもの。これが出れば環境構築成功
同フォルダ/env.pkl・meta_info.json 構築したシーンの全状態スナップショットと、実行時設定の記録
ステップB（タスク生成）の出力 — data/generated_task_from_description/<タスク>_<ID>_<日時>/
config.yaml GPT が生成したタスク定義 YAML。例題と同じ形式（オブジェクト・サイズ・初期関節角・タスク説明）。次のステップC の入力になる
task_<タスク>/substeps.txt ほか定義一式・報酬 .py GPT が生成したサブステップ分解・実行方式・行動空間・報酬関数コード。「LLM がロボットの練習メニューを書く」に相当する RoboGen の核心部分
同フォルダ内の *.json（応答ログ） GPT とのやり取り（プロンプト・応答・所要時間）の記録。生成結果の検証・再現に使える
ステップC（学習）の出力
<solution_path>/primitive_states/<日時>/<サブステップ>/state_*.pkl・execute.gif モーションプランニングの結果。把持完了までの各時刻の環境状態（pkl）と実行映像。最後の state が次サブステップの開始点
<solution_path>/RL_sac/<日時>/<サブステップ>/best_model.zip・latest_model.zip 学習済み方策（本システムの最重要成果物）。SB3 形式のニューラルネット。SAC.load() で読み込めば再利用・追加学習・デモ再生ができる。latest はセッション切れ対策の最新版
同フォルダ/eval_1〜N/execute.gif・state_*.pkl 5000 ステップごとの評価ロールアウト映像と軌跡。学習が進む様子を時系列で確認できる
同フォルダ/best_state/best.gif・state_*.pkl・return_<値>.txt 最良評価時の映像・軌跡・リターン値（例 return_-1933.098.txt）。ファイル名の数値がそのままスコア
<solution_path>/all-<日時>.gif / .mp4 最終成果動画。全サブステップ（把持→操作）を連結した通し映像。セル10 の最後に RESULT_MP4= として表示される
<solution_path>/experiment/meta_info_<日時>.json・all_last_state_files_<日時>.txt 実行設定と、各サブステップ最終状態ファイルへのパス一覧（再開・デバッグ用）
セル11（永続保存）の出力
Google Drive: MyDrive/robogen_outputs/<日時>/*.mp4 直近の動画を Drive にコピー。Colab の /content はセッション終了で消えるため、残したい成果物はここに置く
